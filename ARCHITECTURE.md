# GEMM 矩阵乘法硬件单元架构文档

本文档基于 `README.md` 中的规格要求，给出可落地实现的 GEMM（`D=A×B+C`）硬件架构设计说明，用于指导 RTL 实现与验证。

---

## 1. 设计目标与约束

### 1.1 功能目标
- 支持计算：`D = A × B + C`
- 数据类型：`A/B/C/D` 均为 FP16
- 矩阵维度：
  - `A(m, k)`
  - `B(k, n)`
  - `C(m, n)`
  - `D(m, n)`

### 1.2 接口目标
- 输入：AXI4 Slave（用于接收矩阵数据与配置）
- 输出：AXI4 Master（用于写回矩阵 `D`）

### 1.3 性能目标
- 峰值目标：1 TOPS @ FP16
- 支持带宽优化与 TOPS/W 优化的扩展路径

---

## 2. 顶层架构概览

建议采用“**DMA + 分块缓存 + 计算阵列 + 写回引擎**”的流水化结构。

```text
            +---------------------------+
AXI4 Slave  |   配置/命令/输入接收模块    |
----------> |  (寄存器 + 命令队列)       |
            +-----------+---------------+
                        |
                        v
            +---------------------------+
            |       调度与控制器         |
            |   (tile循环/状态机/同步)   |
            +---+-------------------+---+
                |                   |
         +------v------+     +------v------+
         | A/B/C DMA读  |     | D DMA写回   |
         | +片上Buffer  |     | +写回合并    |
         +------+-------+     +------+------+
                |                    ^
                v                    |
            +---------------------------+
            |       GEMM 计算核心        |
            |   PE阵列 + 累加 + 偏置C    |
            +---------------------------+
```

---

## 3. 核心模块设计

### 3.1 配置与寄存器模块（CSR）
**职责：** 保存任务参数，触发计算，回报状态。

建议寄存器字段：
- `CTRL`：`start`/`soft_reset`/`irq_en`
- `STATUS`：`busy`/`done`/`err_code`
- `DIM_M/N/K`：矩阵维度
- `ADDR_A/B/C/D`：基地址
- `STRIDE_A/B/C/D`：行步长（字节）
- `TILE_M/N/K`：分块参数

### 3.2 调度与控制器
**职责：** 管理 tile 级循环与流水并行。

建议状态机：
1. `IDLE`：等待 `start`
2. `LOAD`：启动 A/B/C tile 读入
3. `COMPUTE`：触发计算阵列进行 MAC
4. `STORE`：写回 D tile
5. `DONE`：置位 `done`，可触发中断

为提升吞吐，建议采用双缓冲（ping-pong）：
- 当前 tile 在计算时，下一 tile 并行搬运。

### 3.3 片上缓存（SRAM/BRAM）
建议拆分：
- `A_BUF`：缓存 A 的 tile（`Tm × Tk`）
- `B_BUF`：缓存 B 的 tile（`Tk × Tn`）
- `C_BUF`：可选缓存 C tile（`Tm × Tn`）
- `ACC_BUF`：部分和缓存（累加结果）

设计原则：
- 地址连续化以提高 AXI 突发效率
- buffer 宽度与计算阵列并行度匹配
- 可选 bank 化降低访问冲突

### 3.4 计算核心（PE Array）

建议参数化 PE 阵列：`P_M × P_N`。
每个 PE 执行 FP16 乘法与加法累加：
- 计算阶段：`acc += A[i][k] * B[k][j]`
- 融合 C：最终 `D[i][j] = acc + C[i][j]`

可选实现策略：
- 纯 FP16 累加（资源低，误差较大）
- FP32 累加后回写 FP16（精度更优，资源更高）

### 3.5 DMA 与 AXI 访问模块

#### 读路径（A/B/C）
- 支持 INCR burst
- 按 tile 自动生成地址
- 支持步长（stride）与边界处理

#### 写路径（D）
- 支持 burst 写回
- 可选写回合并策略，降低小包传输开销

错误处理建议：
- 地址未对齐
- 越界访问
- 维度非法（0 或超范围）

## 4. 数据流与时序

### 4.1 分块计算流程
对输出 tile `D_tile(Tm×Tn)`：
1. 读入 `C_tile`
2. 对 `k` 维做分块循环：
   - 读 `A_tile(Tm×Tk)`、`B_tile(Tk×Tn)`
   - 在 PE 阵列累加部分和
3. `acc + C_tile` 后写回 `D_tile`

### 4.2 性能估算框架
- 理论峰值：
  - `TOPS = 2 × PE并行MAC数 × Freq`
- 带宽需求（近似）：
  - 每 tile 读取：`A + B + C`
  - 每 tile 写回：`D`
- 优化目标：
  - 计算利用率 > 80%
  - AXI 总线有效载荷占比持续提升

## 5. 参数化建议

建议将以下项参数化，以便适配不同 FPGA/ASIC 资源：
- `PE_M`, `PE_N`
- `TILE_M`, `TILE_N`, `TILE_K`
- `AXI_DATA_WIDTH`
- `BUF_DEPTH_A/B/C/ACC`
- `USE_FP32_ACC`（0/1）

## 6. 验证与可测试性建议

### 6.1 功能验证
- 小规模矩阵（如 `4×4`, `8×8`）与软件参考模型逐元素对比
- 随机维度与随机数据回归
- 边界场景：`m/n/k=1`、非对齐 stride、非整 tile

### 6.2 性能验证
- 统计吞吐（GFLOPS/TOPS）
- 统计 DMA 带宽利用率、burst 长度分布
- 记录计算核空闲周期比例

### 6.3 覆盖率建议
- CSR 读写覆盖
- 状态机转移覆盖
- 异常分支覆盖（越界、非法参数）

---

## 7. 后续实现优先级（建议）
1. 最小可用链路：CSR + 单 tile 计算 + D 写回
2. 增加 tile 循环与双缓冲
3. 完善异常处理和中断
4. 优化带宽与功耗并输出测试报告
